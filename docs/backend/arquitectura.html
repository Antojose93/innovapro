Arquitectura

Estructura del backend en src-tauri

El backend esta construido con Rust sobre Tauri. Su diseno sigue una separacion por capas: comandos Tauri para entrada, servicios para reglas, repositorios para SQL, dominio para contratos de datos y un estado compartido para la conexion a la base de datos.

Arbol funcional

src-tauri/src
├─ main.rs                -> punto de entrada ejecutable
├─ lib.rs                 -> registra modulos y expone run()
├─ app/mod.rs             -> construye Tauri Builder y registra commands
├─ state/mod.rs           -> AppState con metadata y SqlitePool
├─ errors.rs              -> AppError y AppResult
├─ commands/              -> interfaz invocable desde el frontend
├─ services/              -> validaciones y logica de negocio
├─ repositories/          -> consultas SQL y escrituras a SQLite
├─ domain/                -> structs de entrada y salida
└─ db/mod.rs              -> esquema, indices, seeds y migraciones ligeras

Secuencia de arranque

  1. 1. main.rs
    Ejecuta erp_produccion_lib::run().
  2. 2. lib.rs
    Declara modulos y llama a app::build().
  3. 3. app/mod.rs
    Construye el runtime Tauri, crea AppState, lo registra con app.manage(...) y expone todos los commands.
  4. 4. state/mod.rs
    Resuelve el directorio de datos local, crea o abre erp_produccion.sqlite, inicializa el pool y ejecuta db::initialize().

Responsabilidad por carpeta

  • app/: configuracion central de Tauri y registro de endpoints.
  • commands/: adaptadores del protocolo Tauri; reciben parametros y delegan.
  • services/: capa de negocio, validacion y orquestacion.
  • repositories/: lectura y escritura SQL usando SQLx.
  • domain/: DTOs, payloads y snapshots serializables.
  • db/: esquema, indices, datos semilla y migraciones evolutivas.

Decision arquitectonica relevante

El backend no usa un ORM de alto nivel ni migraciones externas. El esquema se controla con SQL directo dentro de db/mod.rs. Eso simplifica despliegue y hace visible la evolucion real de tablas y seeds desde el propio codigo fuente.
