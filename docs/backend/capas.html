Capas y flujo

Como se conectan los modulos del backend

La carpeta src-tauri/src esta organizada para que la UI invoque comandos simples, mientras la validacion y el acceso a datos quedan encapsulados en capas internas.

1. Commands

Entrada Tauri

Reciben parametros serializados desde el frontend y delegan en servicios. No concentran reglas de negocio.

2. Services

Reglas y validacion

Normalizan texto, validan referencias, aplican restricciones y coordinan operaciones compuestas.

3. Repositories

SQL explicito

Ejecutan SELECT, INSERT, UPDATE y DELETE sobre SQLite mediante SQLx y devuelven structs del dominio.

4. Domain

Contratos

Define modelos serializables de entrada y salida como CreateUnitInput, Recipe o MasterDataSnapshot.

5. State + DB

Infraestructura

AppState comparte el pool, y db/mod.rs asegura esquema, indices, seeds y migraciones al iniciar.

Ejemplo: creacion de una materia prima

Command

create_material recibe el payload desde Tauri.

Service

Limpia codigo y nombre, valida unidades, costo inicial y existencia de referencias.

Repository

Inserta en materials y opcionalmente crea el primer registro en material_cost_history.

Respuesta

Devuelve un MasterDataSnapshot actualizado para refrescar la UI.

Patrones observados

  • Los servicios retornan snapshots completos despues de operaciones CRUD para simplificar sincronizacion del frontend.
  • Las validaciones de negocio se ejecutan antes de tocar la base de datos.
  • Los errores se exponen como texto serializable mediante AppError.
  • Las operaciones complejas usan transacciones, por ejemplo al crear materiales con costo inicial o migrar recetas desde BOM historico.

Puntos de atencion

  • El dominio FinishedProduct incluye campos agregados por migracion que no estaban en la definicion inicial de la tabla.
  • La base se autoevoluciona en tiempo de arranque con ALTER TABLE y verificaciones de columnas existentes.
  • Existe compatibilidad retroactiva con datos antiguos gracias a la migracion de product_bom_items hacia recipes y recipe_items.
