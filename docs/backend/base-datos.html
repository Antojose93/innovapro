Base de datos

Modelo entidad relacion en SQLite

El esquema se define en src-tauri/src/db/mod.rs. Se crean tablas base, indices, datos semilla y ajustes de compatibilidad al iniciar la aplicacion.

Vista relacional

units

Catalogo de unidades de medida.

  • PK: id
  • UQ: code
  • name, symbol, created_at

materials

Materias primas con dos referencias a unidades.

  • PK: id
  • UQ: code
  • FK: stock_unit_id -> units.id
  • FK: purchase_unit_id -> units.id

material_cost_history

Historico de costos por material y unidad.

  • PK: id
  • FK: material_id -> materials.id
  • FK: unit_id -> units.id
  • unit_cost, effective_at

finished_products

Productos terminados fabricados por la empresa.

  • PK: id
  • UQ: code
  • FK: unit_id -> units.id
  • FK: recipe_id -> recipes.id
  • category, status, internal_code, commercial_code

recipes

Definen formulacion o composicion de un producto.

  • PK: id
  • UQ: code
  • FK: product_id -> finished_products.id
  • status, version, notes

recipe_items

Detalle de materias primas por receta.

  • PK: id
  • FK: recipe_id -> recipes.id
  • FK: material_id -> materials.id
  • FK: unit_id -> units.id
  • UQ: recipe_id + material_id

unit_conversions

Factores de conversion entre unidades.

  • PK: id
  • FK: from_unit_id -> units.id
  • FK: to_unit_id -> units.id
  • UQ: from_unit_id + to_unit_id

product_bom_items

Tabla historica de BOM usada para migrar productos antiguos hacia recetas.

  • PK: id
  • FK: product_id -> finished_products.id
  • FK: material_id -> materials.id
  • UQ: product_id + material_id

Diagrama ER resumido

units
 ├─< materials.stock_unit_id
 ├─< materials.purchase_unit_id
 ├─< material_cost_history.unit_id
 ├─< finished_products.unit_id
 ├─< recipe_items.unit_id
 ├─< unit_conversions.from_unit_id
 └─< unit_conversions.to_unit_id

materials
 ├─< material_cost_history.material_id
 ├─< recipe_items.material_id
 └─< product_bom_items.material_id

finished_products
 ├─< product_bom_items.product_id
 ├─ recipes.product_id
 └─ recipe_id >─ recipes.id

recipes
 └─< recipe_items.recipe_id

Seeds y datos iniciales

  • Se insertan unidades base como KG, G, L, ML y UND.
  • Se agregan unidades especializadas como GAL, VAR6, PLA6, TUB6 y CAJ5.
  • Se siembran materias primas y su costo inicial en material_cost_history.
  • Se registran conversiones predefinidas como KG -> G, GAL -> L y equivalencias de piezas de 6 metros.

Migraciones embebidas

  • Si faltan columnas en finished_products, se agregan category, status, recipe_id, internal_code y commercial_code.
  • Si recipe_items no tiene unit_id, la columna se crea y se rellena con la unidad de stock del material.
  • Si un producto antiguo tiene BOM historica y no tiene receta asociada, se migra a recipes y recipe_items automaticamente.
