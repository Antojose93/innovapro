Arquitectura

Cómo está compuesto el frontend

El frontend es una aplicación Vue 3 con TypeScript que usa Vuetify para la capa visual, Vue Router para navegación y Pinia para estado global. Corre como aplicación web y también como cliente de escritorio sobre Tauri.

Flujo de arranque

  1. 1. `src/main.ts`
    Crea la aplicación Vue, instancia Pinia, registra router y Vuetify, y monta `App.vue`.
  2. 2. `src/App.vue`
    Actúa como contenedor mínimo y delega toda la estructura visual al `MainLayout`.
  3. 3. `src/layouts/MainLayout.vue`
    Renderiza `v-app`, barra superior, drawer lateral, contenedor principal y `router-view` con una transición de desvanecimiento.

Enrutamiento y navegación

Las rutas se definen en `src/router/routes.ts`. Cada ruta incluye `meta` con `title`, `icon` y `showInNavigation`. El layout transforma estas rutas en elementos del menú lateral. Además, `src/router/index.ts` actualiza el título del documento después de cada navegación.

Reglas observadas

  • Las rutas principales con `showInNavigation: true` aparecen en el drawer lateral.
  • `/produccion` redirige automáticamente a `/produccion/unidades`.
  • Las vistas hijas de Producción no se listan en el menú lateral principal.
  • El título del navegador usa el formato `ERP Produccion | {pantalla}`.

Layout principal

  • Barra superior con botón para abrir/cerrar el menú lateral.
  • Drawer persistente controlado por `useAppStore`.
  • Contenedor `v-main` con `v-container` fluido para alojar las vistas.
  • Transición `fade` en el intercambio de pantallas con `router-view`.

Tema visual

`src/plugins/vuetify.ts` define un tema claro personalizado con fondo `#F8FAFC`, color primario `#5568a5` y colores secundarios para estados de éxito, advertencia e información. También fija defaults de componentes como `VCard`, `VTextField`, `VSelect`, `VBtn` y `VDataTable`.
