Estado y servicios

Flujo de datos del frontend

La aplicación distribuye responsabilidades entre stores de Pinia, tipos TypeScript y un servicio de escritorio que abstrae la comunicación con Tauri o un fallback mock cuando corre como web.

Stores de Pinia

`useAppStore`

Store pequeño enfocado en UI global. Solo administra `isSidebarOpen` y la acción `toggleSidebar()` para el menú lateral.

`useMasterDataStore`

Store principal del frontend. Mantiene `units`, `materials`, `products`, `recipes`, `conversions`, `costHistory`, estados `loading` y `saving`, mensajes de éxito/error y el resultado de conversiones.

Patrón operativo

  • `hydrate()` consulta el snapshot completo y refresca historial de costos si aplica.
  • `withTask()` centraliza control de `loading`, `saving` y captura uniforme de errores.
  • Cada acción CRUD delega en `desktopService` y luego reaplica el snapshot recibido.
  • La vista `Maestros` consume ese store y lo combina con tabs, catálogos y formularios.

Servicio de integración: `desktopService`

`src/services/desktop.ts` concentra la integración del frontend con Tauri. El servicio usa `invoke()` de `@tauri-apps/api/core` y expone métodos para salud del sistema, tabs de producción, snapshot maestro, historial de costos, CRUD de unidades/materiales/productos/ recetas/conversiones y cálculo de conversiones. 

UI Vue
  -> Pinia store
    -> desktopService
      -> isTauriApp()
        -> invoke(...) si corre en Tauri
        -> fallback mock si corre como web

Detección de entorno

`src/lib/tauri.ts` considera que la app está dentro de Tauri si existe `__TAURI_INTERNALS__` en `window`. Si no existe, el frontend trabaja con datos mock locales.

Fallback mock

El frontend puede funcionar sin backend activo. `desktopService` incluye un `desktopFallback` con catálogo inicial de unidades, materias primas, productos, recetas, conversiones e historial de costos. Esto permite desarrollar la interfaz y validar flujos sin depender de Tauri.

También resuelve conversiones entre unidades por recorrido de grafo, no solo por coincidencias directas, lo que amplía la capacidad del mock para pruebas de la UI.
