Pantallas

Módulos documentados del frontend

El sistema presenta un shell principal con cinco entradas de navegación. Algunas ya ofrecen comportamiento CRUD real sobre datos maestros y otras operan todavía como pantallas base o de visualización.

Dashboard Vista principal

Resumen ejecutivo y estado del entorno

  • Muestra cuatro KPI de ejemplo: ventas, órdenes activas, stock crítico y cuentas por cobrar.
  • Renderiza una tabla con las últimas cinco órdenes usando datos mock locales.
  • Ejecuta `desktopService.healthCheck()` al montar la vista.
  • Si no se ejecuta dentro de Tauri, reporta un estado `detached` con entorno `browser`.
Maestros Módulo CRUD principal

Gestión central de datos maestros

Es la vista más completa del frontend actual. Usa tabs, catálogos, formularios modales, snackbar de éxito, alertas de error y acciones de refresco sobre el store `masterData`.

Materias primas

Catálogo, edición, costos e historial.

Unidades

Alta, edición y eliminación de unidades base.

Recetas

Definición de fórmulas e insumos por receta.

Productos

Productos terminados y asociación con recetas.

Conversiones

Inventario

Actualmente funciona como una pantalla descriptiva. Presenta una lista visual de áreas del módulo: almacenes, stock actual, movimientos y alertas.

Finanzas

También es una vista base. Enumera cuentas por cobrar, cuentas por pagar, bancos y reportes financieros, sin lógica de datos adicional por ahora.

Producción

Es un módulo contenedor con tabs internos generados desde `desktopService.listProductionTabs()` y un `router-view` hijo para separar subpantallas.

Submódulos de Producción

Unidades

Vista demostrativa con catálogo base y lista de acciones. Muestra feedback mediante snackbar, pero todavía conserva comentarios `TODO` para conectar navegación o lógica real.

Recetas

Usa el store de maestros para listar recetas, ingredientes, estado, producto asociado y métricas de seguimiento. Tiene tablas y paneles expandibles.

Productos

Muestra indicadores de productos activos, promedio de insumos por receta, tabla de productos y un panel de control de recetas asociadas.
